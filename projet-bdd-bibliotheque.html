Base de Données pour la Gestion des Championnats de Football

Informations du projet

  • Date : Novembre 2022 - Janvier 2023
  • Durée : 6 semaines
  • Contexte : Projet universitaire
  • Compétence : Gérer des données de l'information

Technologies utilisées :

MySQL Tutorial D UML Visual Paradigm

Description du projet

Conception et implémentation d'une base de données relationnelle pour la gestion des résultats des championnats de football français (Ligue 1, Ligue 2, Division 1 féminine, etc.), incluant clubs, équipes, joueurs, matches et statistiques.

Schéma conceptuel

Livrables produits

  • Diagramme de classe UML complet avec règles de gestion
  • Script Tutorial D (.rel) pour création automatique du schéma
  • Graphe des contraintes d'intégrité référentielles
  • Script SQL de création de la base de données
  • Requêtes SQL répondant aux besoins métiers

Fonctionnalités implémentées

  • Gestion des clubs et équipes (acronymes, entraîneurs)
  • Suivi des saisons et journées de championnat
  • Enregistrement des matches (dates, résultats, affluence)
  • Gestion des joueurs (statistiques, compositions d'équipe)
  • Suivi des cartons et temps de jeu

Exemple de requête SQL

Classement d'une ligue pour une saison donnée

SELECT e.nom AS equipe, 
       COUNT(CASE WHEN m.score_local > m.score_visiteur THEN 1 END) * 3 + 
       COUNT(CASE WHEN m.score_local = m.score_visiteur THEN 1 END) AS points,
       COUNT(CASE WHEN m.score_local > m.score_visiteur THEN 1 END) AS victoires,
       COUNT(CASE WHEN m.score_local = m.score_visiteur THEN 1 END) AS nuls,
       COUNT(CASE WHEN m.score_local < m.score_visiteur THEN 1 END) AS defaites,
       SUM(m.score_local) AS buts_pour,
       SUM(m.score_visiteur) AS buts_contre
FROM Equipes e
LEFT JOIN Matches m ON (e.id = m.equipe_local_id OR e.id = m.equipe_visiteur_id)
WHERE m.saison_id = 12
GROUP BY e.id
ORDER BY points DESC, (buts_pour - buts_contre) DESC, buts_pour DESC;
